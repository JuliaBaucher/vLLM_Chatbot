Implementation Overview ▾
This page is a static, single-file web application. When the user sends a message, the browser makes a POST /chat request to a backend API. The backend receives the message, applies the system prompt and application logic, and calls a local vLLM server through its OpenAI-compatible API. vLLM then runs the open-source model and returns the generated response to the backend, which sends a JSON reply back to the browser.
• Frontend: static HTML/CSS/JS chat UI running in the browser
• Backend: receives user messages and calls the model server
• Model server: vLLM exposes an OpenAI-compatible local API
• Prompt logic lives in the backend, not in the frontend
• This prototype is stateless unless conversation history is added in backend code
• localStorage is used only for browser-side chat persistence
• The frontend renders messages as plain text to reduce XSS risk
